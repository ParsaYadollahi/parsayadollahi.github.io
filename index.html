  • HackPrinceton

    November 2018

    I Participated in my first Hackathon at Princeton in the Fall 2018 semester where me and two of my classmates at McGill worked on a project called CourseTalk. The motivation behind this project is the lack of engagement in school discussion boards. The issue with these panels is the permanence formality of having to submit posts, which implies that most students resort to Facebook group chats. The issue with Facebook is the unawareness that students have to these group chats and the distractions leaded by Facebook. Which gave us the inspiration to create a platform similar to that of Slack, which is not as formal as discussion panels and that would be integrated into students' default course page. This would lead to a simpler communication between students about their coursework.

Project

Functionality

CourseTalk connects students of a given university with courses in common by automatically adding them to group conversations restricted to said course. This creates a space where everyone is comfortable to ask their questions, brainstorm share lecture notes or even arrange real-life study groups, all distraction-free.

Story Time

CourseTalk was build using React.js. We were new to JavaScript and React so the first half of the hackathon (about 10 hours) was spent on watching and following online Tutorials to build the project. The tutorial that we were following used two instances of Node.js as its backend. The first to run the web application on Express.js and the second to support the React frontend. Which implies that the website would only be supported on our localhost server which supplied two ports but would not work on GitHub pages. After hours of try and failure to make it work with multiple ports, we had a friend from a different that suggested to port the project to Webpack and to use Heroku to host the app. With this, the app was up on the internet. Finally, we pushed the project to Heroku but it did not work. We found that the app would only work without the CSS component, but by the time we figured that out, the hackathon was very close to its deadline. Thus, we created a stable branch with the old CSS which was posted to Heroku and a master branch with the new CSS (which is the first image presented to the right).
